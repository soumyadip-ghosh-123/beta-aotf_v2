## Overview

The `Job` model represents a job or freelance project listing posted by an admin on behalf of a client. It supports both traditional employment and project-based work.

**Collection:** `jobs`
**File:** `lib/models/Job.ts`

## Enums

### Work Type

| Value | Description |
|-------|-------------|
| `job` | Traditional employment position |
| `project` | Freelance project work |

### Company Type

| Value | Description |
|-------|-------------|
| `individual` | Individual client |
| `company` | Corporate/company client |

### Location Type

| Value | Description |
|-------|-------------|
| `remote` | Fully remote |
| `onsite` | On-site work |
| `hybrid` | Mix of remote and on-site |

### Gender Preference

| Value | Description |
|-------|-------------|
| `male` | Male preferred |
| `female` | Female preferred |
| `both` | Either gender |
| `all` | No preference |

### Commission Basis

| Value | Used For | Calculation |
|-------|----------|-------------|
| `first_month` | Jobs | Commission on first month's salary |
| `project_value` | Projects | Commission on total project value |

## Status Flow

| Status | Description |
|--------|-------------|
| `open` | Actively accepting applications |
| `closed` | Position filled or completed |
| `hold` | Temporarily paused |
| `cancelled` | Permanently cancelled |
