## Overview

The `Post` model represents a tuition requirement posted by an admin on behalf of a guardian/parent. It tracks student details, scheduling preferences, budget, and lifecycle status.

**Collection:** `posts`
**File:** `lib/models/Post.ts`

---

## Status Flow

| Status | Description |
|--------|-------------|
| `open` | Actively accepting applications |
| `matched` | Teacher has been assigned |
| `closed` | Tuition completed/ended |
| `cancelled` | Post was cancelled |
| `hold` | Temporarily paused |

### Payment Status

| Status | Description |
|--------|-------------|
| `done` | Guardian payment received |
| `pending` | Payment pending |

## Indexes

| Fields | Purpose |
|--------|---------|
| `postId: 1` | Unique lookup |
