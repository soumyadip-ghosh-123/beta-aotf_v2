## System Requirements

Ensure you have the following installed on your machine:

| Requirement | Minimum Version | Check Command |
|------------|----------------|---------------|
| **Node.js** | 18.x or later | `node --version` |
| **pnpm** | 11.x | `pnpm --version` |
| **Git** | 2.x | `git --version` |

### Installing pnpm

If you don't have pnpm installed:

```bash
npm install -g pnpm
```

## Required Accounts

You'll need accounts (free tiers are sufficient for development) on the following services:

| Service | Purpose | Sign Up |
|---------|---------|---------|
| **MongoDB Atlas** | Database hosting | [mongodb.com/atlas](https://www.mongodb.com/atlas) |
| **Clerk** | Authentication | [clerk.com](https://clerk.com) |
| **Razorpay** | Payment gateway | [razorpay.com](https://razorpay.com) |
| **Cloudinary** | Media uploads | [cloudinary.com](https://cloudinary.com) |
| **Resend** | Transactional emails | [resend.com](https://resend.com) |
| **Sentry** | Error monitoring | [sentry.io](https://sentry.io) |

> **Note**: For local development, you can skip Sentry and Vercel Analytics. The app will run without them.

## Clerk Configuration

Your Clerk instance should have the following enabled:

- ✅ **Username authentication** enabled
- ✅ **Password authentication** enabled
- ✅ **Email** enabled (for notifications)
- ✅ **Webhook** configured to point to `/api/v1/webhooks/clerk`

The same Clerk instance is used for both regular users and admins. Differentiation is done via `publicMetadata.isAdmin`.

---

## MongoDB Atlas Setup

1. Create a free M0 cluster
2. Add your IP address to the network access list (or use `0.0.0.0/0` for development)
3. Create a database user
4. Copy the connection string — you'll need it for `MONGODB_URI`

> **Tip**: AOTF handles Atlas M0 cold starts with retry logic and exponential backoff. Your free cluster will work fine.
