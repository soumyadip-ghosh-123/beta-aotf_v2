## Setup

Create a `.env.local` file in the project root:

```bash
cp .env.example .env.local  # if an example exists
# or create manually
touch .env.local
```

---

## Required Variables

### Clerk Authentication

```bash
# Clerk publishable key (client-side)
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_...

# Clerk secret key (server-side)
CLERK_SECRET_KEY=sk_test_...

# Clerk webhook signing secret
CLERK_WEBHOOK_SECRET=whsec_...

# Sign-in/up redirect URLs
NEXT_PUBLIC_CLERK_SIGN_IN_URL=/sign-in
NEXT_PUBLIC_CLERK_SIGN_UP_URL=/sign-up
NEXT_PUBLIC_CLERK_AFTER_SIGN_IN_URL=/
NEXT_PUBLIC_CLERK_AFTER_SIGN_UP_URL=/onboarding
```

### MongoDB

```bash
# MongoDB Atlas connection string
MONGODB_URI=mongodb+srv://username:password@cluster.mongodb.net/aotf?retryWrites=true&w=majority
```

### Application URL

```bash
# Your app's public URL (used for CSRF, emails, etc.)
NEXT_PUBLIC_APP_URL=http://localhost:3000
NEXT_PUBLIC_SITE_URL=http://localhost:3000
```

## Payment & Finance

### Razorpay

```bash
# Razorpay API keys
RAZORPAY_KEY_ID=rzp_test_...
RAZORPAY_KEY_SECRET=...

# Public key for client-side checkout
NEXT_PUBLIC_RAZORPAY_KEY_ID=rzp_test_...
```

## Media & Communication

### Cloudinary

```bash
# Cloudinary configuration
NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME=your-cloud-name
CLOUDINARY_API_KEY=...
CLOUDINARY_API_SECRET=...
```

### Resend (Email)

```bash
# Resend API key for transactional emails
RESEND_API_KEY=re_...

# Sender email address (must be verified in Resend)
EMAIL_FROM="AOTF <noreply@yourdomain.com>"
```

### Google Sheets (Optional)

```bash
# Service account credentials file path
GOOGLE_APPLICATION_CREDENTIALS=./aotf-service-account.json

# Spreadsheet ID for enquiry ledger
GOOGLE_SHEET_ID=...
```

## Monitoring (Optional)

### Sentry

```bash
# Sentry DSN for error tracking
SENTRY_DSN=https://...@sentry.io/...
SENTRY_AUTH_TOKEN=sntrys_...
SENTRY_ORG=aotf
SENTRY_PROJECT=javascript-nextjs
```

> **Note**: Sentry is configured via `sentry.server.config.ts` and `sentry.edge.config.ts`. The monitoring tunnel route is `/monitoring`.

## Variable Reference Table

| Variable | Required | Client | Description |
|----------|----------|--------|-------------|
| `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` | ✅ | ✅ | Clerk public key |
| `CLERK_SECRET_KEY` | ✅ | ❌ | Clerk server key |
| `CLERK_WEBHOOK_SECRET` | ✅ | ❌ | Webhook signing |
| `MONGODB_URI` | ✅ | ❌ | Database connection |
| `NEXT_PUBLIC_APP_URL` | ✅ | ✅ | App base URL |
| `RAZORPAY_KEY_ID` | ✅ | ❌ | Payment key |
| `RAZORPAY_KEY_SECRET` | ✅ | ❌ | Payment secret |
| `NEXT_PUBLIC_RAZORPAY_KEY_ID` | ✅ | ✅ | Client payment key |
| `NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME` | ✅ | ✅ | Cloudinary cloud |
| `RESEND_API_KEY` | ✅ | ❌ | Email API key |
| `EMAIL_FROM` | ✅ | ❌ | Sender address |
| `SENTRY_DSN` | ❌ | ❌ | Error tracking |
| `GOOGLE_SHEET_ID` | ❌ | ❌ | Sheets sync |

> Variables prefixed with `NEXT_PUBLIC_` are exposed to the browser. Never put secrets in `NEXT_PUBLIC_` variables.
