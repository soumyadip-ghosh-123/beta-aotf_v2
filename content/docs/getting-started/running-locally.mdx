## Start the Dev Server

Once you've configured your [environment variables](/docs/getting-started/environment-variables), start the development server:

```bash
pnpm dev
```

This runs `next dev --turbopack`, which uses **Turbopack** for fast module bundling. The server starts on `http://localhost:3000` by default.

---

## What Happens on Startup

When the dev server starts, several things happen automatically:

1. **Database Connection Warmup** — The `instrumentation.ts` file initiates a MongoDB connection so the first user request gets a hot connection.
2. **Fumadocs Source Generation** — The `.source` directory is generated from `content/docs/` MDX files.
3. **Sentry Initialisation** — Error tracking hooks are registered (if configured).

## Verify Your Setup

### 1. Homepage

Navigate to [http://localhost:3000](http://localhost:3000). You should see the AOTF landing page.

### 2. Documentation

Navigate to [http://localhost:3000/docs](http://localhost:3000/docs). You should see this documentation site.

### 3. Admin Login

Navigate to [http://localhost:3000/admin/login](http://localhost:3000/admin/login). This should show the admin sign-in page.

### 4. API Health Check

```bash
curl http://localhost:3000/api/v1/posts
```

This should return a JSON response (empty array or posts list).

## Create the Superadmin

If this is a fresh setup, create the initial superadmin account:

```bash
pnpm create:admin
```

This runs `tsx scripts/create-admin.ts` and outputs:


## Common Issues

### MongoDB Connection Timeout

If you see `MongoServerSelectionError`:

- Verify your `MONGODB_URI` is correct
- Check that your IP is whitelisted in Atlas
- The app retries up to 3 times with exponential backoff (2s → 4s → 8s)

### Clerk Authentication Errors

- Ensure both `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` and `CLERK_SECRET_KEY` are set
- Verify the keys match the same Clerk instance
- Check the Clerk dashboard for any configuration issues

### Port Already in Use

```bash
# Find and kill the process on port 3000
npx kill-port 3000

# Or use a different port
pnpm dev -- --port 3001
```

## Production Build

To test a production build locally:

```bash
pnpm build
pnpm start
```

The build process includes:
- Fumadocs MDX compilation
- Static page generation
- Sentry source map upload (if configured)
