# Manage Users in Admin

The **Users** page at `/admin/users` is the operational hub for provider accounts after [legacy migration](/docs/how-to/migrate-legacy-users) or day-to-day admin work.

**Permission required**: `canManageUsers` (or `super_admin`).

## Data Source

The page lists users from **MongoDB** (`users` + `profiles`), enriched with Clerk email and avatar. It does **not** query Clerk as the primary store.

On every load, the page calls:

```http
GET /api/admin/app-users?bundle=1&sync=1
```

The `sync=1` flag runs `syncClerkAppUsers()` first — creating any missing Mongo records for Clerk accounts. This is why migrated users appear here even if the migration script's Mongo seed step failed partway.

## UI Features

| Feature | Behavior |
|---|---|
| **Teachers / Candidates tabs** | Filter by `role` (`teacher` vs `teacher_candidate`) |
| **Status filter** | `active`, `blocked`, `deleted`, or all |
| **Search** | Username or email |
| **Pagination** | 10 users per page |
| **Refresh** | Re-runs sync + reload |
| **Profile link** | Opens `/u/[username]` |
| **Verify link** | `/verify/AOTF-T-{USERNAME}` or `AOTF-C-...` for candidates |

## Create User (FAB)

On `/admin/users`, the floating action button opens a **Create User** modal:

- Name, email, phone, role (teacher or candidate)
- `POST /api/admin/app-users`
- Creates Clerk account without password; user signs in via Google or sets password

## Block / Unblock / Delete

Status actions call `PATCH /api/admin/app-users/:userId/status`:

- **Block** — revokes Clerk sessions; user cannot sign in
- **Unblock** — restores `active` status
- **Delete** — soft delete in MongoDB; sessions revoked

All actions are logged in the activity log.

## Troubleshooting

| Symptom | Fix |
|---|---|
| User in Clerk but not in admin list | Click Refresh (runs sync) or `pnpm sync:clerk-users` |
| Migration completed but counts mismatch | Run [sync script](/docs/how-to/seed-users#post-migration-sync) with `--live` |
| User cannot sign in after block | Unblock via status API or admin UI |
