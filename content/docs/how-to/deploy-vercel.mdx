# Deploy to Vercel

AOTF runs on **Vercel's free Hobby tier** for hosting and **MongoDB Atlas M0** (free) for the database. This guide covers the full deployment setup.

## Prerequisites

- Vercel account (hobby tier is free)
- MongoDB Atlas account with an M0 cluster
- Clerk account (development or production instance)
- Razorpay account (test or live mode)

## 1. Connect Repository to Vercel

1. Go to [vercel.com/new](https://vercel.com/new)
2. Import your Git repository
3. Vercel auto-detects Next.js — no build config needed

The `vercel.json` in the project root sets:

```json
{
  "framework": "nextjs"
}
```

## 2. Set Environment Variables

In the Vercel project dashboard → **Settings → Environment Variables**, add all required variables:

### Required for All Environments

```bash
# MongoDB
MONGODB_URI=mongodb+srv://...

# Clerk
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_live_...
CLERK_SECRET_KEY=sk_live_...
CLERK_WEBHOOK_SECRET=whsec_...

# App
NEXT_PUBLIC_APP_URL=https://aotf.in

# Razorpay
RAZORPAY_KEY_ID=rzp_live_...
RAZORPAY_KEY_SECRET=...
NEXT_PUBLIC_RAZORPAY_KEY_ID=rzp_live_...
RAZORPAY_WEBHOOK_SECRET=...

# Cloudinary
NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME=...
CLOUDINARY_API_KEY=...
CLOUDINARY_API_SECRET=...

# Google Sheets
GOOGLE_SERVICE_ACCOUNT_EMAIL=...@....iam.gserviceaccount.com
GOOGLE_SERVICE_ACCOUNT_PRIVATE_KEY="-----BEGIN RSA PRIVATE KEY-----\n..."
GOOGLE_SHEETS_SPREADSHEET_ID=...

# Resend (email)

# Sentry
NEXT_PUBLIC_SENTRY_DSN=https://...@sentry.io/...
SENTRY_AUTH_TOKEN=...
SENTRY_ORG=...

```

See the [Environment Variables reference](/docs/reference/env-vars) for complete descriptions.

## 3. Vercel Free Tier Limits

| Limit | Value | Impact |
|---|---|---|
| Bandwidth | 100 GB / month | Sufficient for moderate traffic |
| Serverless function timeout | 10 seconds | DB queries + retries must complete in < 10s |
| Serverless function memory | 1024 MB | Adequate for current workload |
| Deployments | Unlimited | CI/CD deploys on every push |
| Custom domains | 1 (Hobby) | `aotf.in` + `docs.aotf.in` via CNAME |

> **⚠ Cold start warning**: Vercel serverless functions cold-start on the first request after inactivity. Combined with MongoDB M0 auto-pause, the first request after a quiet period can take **5–15 seconds**. See [Cold Starts & Free Tier](/docs/explanations/cold-start) for how the codebase handles this.

## 4. Custom Domain Setup

1. In Vercel → **Settings → Domains**, add `aotf.in`
2. Add a CNAME record in your DNS: `aotf.in → cname.vercel-dns.com`
3. For `docs.aotf.in`, add another CNAME pointing to the same Vercel project

The `proxy.ts` file handles the `docs.aotf.in` host routing:

```ts
// proxy.ts — auto-redirects docs.aotf.in/ to /docs
const isDocsHost = host === "docs.aotf.in" || host === "www.docs.aotf.in";
if (isDocsHost && pathname === "/") {
  return NextResponse.redirect(new URL("/docs", req.url));
}
```

## 5. Configure Clerk Webhook

After deployment, register the Clerk webhook endpoint:

1. Clerk Dashboard → **Webhooks → Add Endpoint**
2. URL: `https://aotf.in/api/v1/webhooks/clerk`
3. Events to subscribe: `user.created`, `user.updated`, `user.deleted`
4. Copy the **Signing Secret** → paste as `CLERK_WEBHOOK_SECRET` in Vercel

## 6. Configure Razorpay Webhook

1. Razorpay Dashboard → **Settings → Webhooks → Add New Webhook**
2. URL: `https://aotf.in/api/v1/webhooks/razorpay`
3. Events: `payment.captured`, `payment.failed`
4. Copy the **Webhook Secret** → paste as `RAZORPAY_WEBHOOK_SECRET` in Vercel

## 7. Verify Deployment

```bash
# Health check (should return 200)
curl https://aotf.in/api/health

# Expected response:
# { "status": "ok", "db": "connected", "uptime": ..., "version": "0.0.1" }
```

## 8. MongoDB Atlas M0 Configuration

### Connection String Format

```
mongodb+srv://<user>:<password>@<cluster>.mongodb.net/<database>?retryWrites=true&w=majority
```

### Allow Vercel IPs

MongoDB Atlas M0 does not support IP whitelisting — you must set **Network Access → Allow Access from Anywhere** (`0.0.0.0/0`). This is a known limitation of the M0 free tier.

### Auto-Pause Behaviour

The M0 cluster **auto-pauses after 60 minutes of inactivity**. The `dbConnect()` function in `lib/db.ts` handles this with:

- `serverSelectionTimeoutMS: 15_000` — gives the cluster 15s to wake up
- Exponential retry: up to **3 attempts**, with delays of 2s → 4s → 8s
- `instrumentation.ts` runs on server startup to pre-warm the connection

See [Cold Starts & Free Tier](/docs/explanations/cold-start) for a full explanation.
