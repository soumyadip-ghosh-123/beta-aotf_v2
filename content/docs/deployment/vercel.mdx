## Overview

AOTF is designed for deployment on **Vercel**, leveraging its serverless functions, edge middleware, and CDN for optimal performance.

---

## Deployment Steps

### 1. Connect Repository

1. Go to [vercel.com](https://vercel.com) and create an account
2. Click "New Project" and import your Git repository
3. Vercel automatically detects the Next.js framework

### 2. Configure Environment Variables

Add all required environment variables in the Vercel dashboard:

**Settings → Environment Variables**

> See [Environment Variables](/docs/getting-started/environment-variables) for the full list.

### 3. Configure Build Settings

Vercel auto-detects these, but verify:

| Setting | Value |
|---------|-------|
| Framework | Next.js |
| Build Command | `pnpm build` |
| Output Directory | `.next` |
| Install Command | `pnpm install` |
| Node.js Version | 18.x or later |

### 4. Deploy

Push to your main branch or trigger a manual deployment. Vercel will:

1. Install dependencies with pnpm
2. Run `next build` (which triggers Fumadocs MDX compilation)
3. Upload source maps to Sentry (if configured)
4. Deploy serverless functions and static assets

## Vercel Configuration

The project includes a `vercel.json` file:

```json
{
  "framework": "nextjs"
}
```

## Domain Configuration

### Main Domain

Configure your primary domain (e.g., `aotf.in`) in Vercel's domain settings.

### Docs Subdomain

AOTF supports a docs subdomain (`docs.aotf.in`) that redirects to `/docs`:

1. Add `docs.aotf.in` as a domain in Vercel
2. The middleware handles the redirect automatically:

```typescript
const isDocsHost = host === "docs.aotf.in" || host === "www.docs.aotf.in";
if (isDocsHost && pathname === "/") {
  return NextResponse.redirect(new URL("/docs", req.url));
}
```

## Performance Considerations

### Serverless Function Limits

| Resource | Vercel Limit (Hobby) | Vercel Limit (Pro) |
|----------|---------------------|-------------------|
| Execution Duration | 10s | 60s |
| Memory | 1024 MB | 3008 MB |
| Payload Size | 4.5 MB | 4.5 MB |

### Cold Start Mitigation

AOTF handles serverless cold starts through:

1. **DB Connection Warmup** — `instrumentation.ts` initiates a MongoDB connection at startup
2. **Connection Pooling** — Shared pool with `maxPoolSize: 10`
3. **Retry Logic** — 3 retries with exponential backoff for database connections

### Bundle Size

The Sentry config optimises bundle size:

```javascript
treeshake: {
  removeDebugLogging: true, // Remove debug statements
}
```
