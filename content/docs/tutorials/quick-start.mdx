# Quick Start (Developers)

This guide gets you from zero to a running local dev server. It assumes you have **Node.js 20+**, **pnpm 11+**, and **Git** installed.

## 1. Clone the Repository

```bash
git clone <repo-url> aotf_v2
cd aotf_v2
```

## 2. Install Dependencies

The project uses **pnpm** as the package manager (enforced via `packageManager` in `package.json`).

```bash
pnpm install
```

> **Note**: The `.npmrc` file configures pnpm hoisting rules. Do not switch to npm or yarn — it may break module resolution.

## 3. Configure Environment Variables

Copy the example env file and fill in the required values:

```bash
cp .env.example .env.local
```

> If `.env.example` doesn't exist, create `.env.local` manually. See the [Environment Variables reference](/docs/reference/env-vars) for all required keys.

### Minimum Required Variables

```bash
# MongoDB
MONGODB_URI=mongodb+srv://<user>:<password>@<cluster>.mongodb.net/<db>?retryWrites=true&w=majority

# Clerk
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_...
CLERK_SECRET_KEY=sk_test_...
CLERK_WEBHOOK_SECRET=whsec_...

# App URL
NEXT_PUBLIC_APP_URL=http://localhost:3000

# Razorpay (test mode)
RAZORPAY_KEY_ID=rzp_test_...
RAZORPAY_KEY_SECRET=...
NEXT_PUBLIC_RAZORPAY_KEY_ID=rzp_test_...
```

## 4. Start the Dev Server

```bash
pnpm dev
```

The app starts on **[http://localhost:3000](http://localhost:3000)** using Next.js Turbopack for fast hot-reloads.

```
▲ Next.js 16.1.6 (Turbopack)
- Local:        http://localhost:3000
- Ready in 2.1s
```

## Common Issues

### MongoDB connection fails locally

Some ISPs can't resolve Atlas cluster hostnames. The `lib/db.ts` file overrides DNS to `1.1.1.1` in development:

```ts
// lib/db.ts — already handled for you
if (process.env.NODE_ENV !== "production") {
  dns.setServers(["1.1.1.1"]);
}
```

If you still see DNS errors, check your VPN or try a different network.

### Clerk webhook not working locally

Use [ngrok](https://ngrok.com/) or [Clerk's local webhooks](https://clerk.com/docs/integrations/webhooks/sync-data) to expose your local server:

```bash
ngrok http 3000
# Then update CLERK_WEBHOOK_SECRET in Clerk dashboard with the ngrok URL
```

### Port 3000 already in use

```bash
# Find and kill the process using port 3000
npx kill-port 3000
```
