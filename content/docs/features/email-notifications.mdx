## Overview

AOTF uses **Resend** for transactional email delivery. The email service (`lib/services/email.service.ts`) provides templated email functions for admin and user notifications.

---

## Configuration

```bash
# .env.local
RESEND_API_KEY=re_...
EMAIL_FROM="AOTF <noreply@yourdomain.com>"
```

> The "From" email address must be verified in the Resend dashboard.

---

## Email Triggers

### Admin Notifications

| Event | Recipient | Description |
|-------|-----------|-------------|
| Account Created | New admin | Welcome email with temporary password |
| Password Reset | Admin | Notification that password was reset by Super Admin |
| Account Locked | Admin | Alert that account was locked after 5 failed attempts |
| Account Unlocked | Admin | Notification that account was unlocked |

### User Notifications

| Event | Recipient | Description |
|-------|-----------|-------------|
| Application Submitted | User | Confirmation of application |
| Application Approved | User | Notification of selection |
| Application Declined | User | Notification with decline reason |
| Onboarding Complete | User | Welcome after completing registration |

## Email Service Pattern

```typescript
// lib/services/email.service.ts

import { Resend } from "resend";

const resend = new Resend(process.env.RESEND_API_KEY);

export async function sendAdminCreatedEmail(params: {
  email: string;
  username: string;
  temporaryPassword: string;
}) {
  await resend.emails.send({
    from: process.env.EMAIL_FROM,
    to: params.email,
    subject: "Your AOTF Admin Account",
    html: `
      <h2>Welcome to AOTF Admin</h2>
      <p>Your admin account has been created.</p>
      <p><strong>Username:</strong> ${params.username}</p>
      <p><strong>Temporary Password:</strong> ${params.temporaryPassword}</p>
      <p>Please change your password on first login.</p>
    `,
  });
}
```

## Error Handling

Email sending failures are logged but don't block the primary operation:

```typescript
try {
  await sendAdminCreatedEmail({ ... });
} catch (error) {
  console.error("[Email] Failed to send:", error);
  // Don't throw — the admin was still created successfully
}
```

## Troubleshooting

### Emails Not Sending

1. Verify `RESEND_API_KEY` is set and valid
2. Check that `EMAIL_FROM` is configured
3. Ensure the "From" email address is verified in the Resend dashboard
4. Check Resend dashboard logs for delivery status
5. Review server logs for error messages

### Rate Limits

Resend has rate limits based on your plan. For development, the free tier allows 100 emails/day and 1 email/second.
