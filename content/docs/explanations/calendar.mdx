# Calendar System

The admin **Unified Calendar** aggregates deadlines and scheduled actions from across the platform into a single view.

## Data Model

Calendar events are stored in the `CalendarEvent` collection. See the field reference in [Database — Calendar](/docs/reference/db/calendar).

## Event Sources

| Source | Trigger |
|---|---|
| Applications | `dcDate`, `gcDate`, status changes |
| Enquiries | Follow-up dates |
| Feedback | Scheduled review dates |
| Admin todos | Manual `TodoEvent` entries |

Mongoose hooks on `Application` and related models upsert `CalendarEvent` records when dates change.

## Backfill

After data migration or bulk imports, rebuild calendar events:

```bash
pnpm tsx scripts/backfill-calendar-events.ts
```
