# Account Linking

AOTF supports **multiple sign-in methods on one Clerk account** — primarily Google SSO and username/password. This matters most for [migrated legacy users](/docs/how-to/migrate-legacy-users), who are created without passwords.

## Migrated Users (Google First)

Migration scripts create Clerk users with:

- Email address from legacy MongoDB
- No password (`skipPasswordRequirement: true`)
- `migratedFromLegacy: true` in `publicMetadata`

These users should sign in with **Continue with Google** using the **same Gmail** as their migrated email. Clerk links the Google OAuth identity to the existing account automatically.

Alternatively:

- **Forgot password** on the sign-in page — sets a password on first use
- Same result: one account, two sign-in methods

## Link Password After Google Sign-In

If a user signs in with Google and wants to add password login:

1. Visit `/verify/link-account` (or follow the in-app prompt)
2. Set a new password while signed in
3. Future sign-ins work with either Google or username/password

The sign-in page shows a banner when opened with `?mode=link-account`:

> This account already exists. Sign in to the same account, and we will guide you to add password login as an additional sign-in method.

## New Self-Service Sign-Ups

Users who register at `/sign-up` create accounts with **username + email + password** first. They may later connect Google from Clerk's account settings if enabled.

## Clerk Configuration

- **Production**: Enable Google for migrated users — see [Configure Clerk](/docs/how-to/configure-clerk)
- **Development**: Google optional; test with password-based accounts
