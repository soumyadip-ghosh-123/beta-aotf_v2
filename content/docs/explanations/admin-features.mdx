# Admin Features

The AOTF admin panel is a comprehensive CRM/FRM built to handle the entire lifecycle of a tutoring/freelance business.

## 1. Role-Based Access Control (RBAC)

Not all admins are equal. The system enforces strict access control:
- A `crm` admin can handle enquiries and create posts, but cannot see financial data.
- A `super_admin` can do everything, including managing other admins and issuing refunds.
- Actions are gated in the UI (buttons hide if you lack permission) and enforced at the API layer (requests are rejected if you bypass the UI).

## 2. Analytics Dashboard

Accessible only to admins with the `canViewAnalytics` permission. It aggregates real-time data across the platform:
- **Pipeline Metrics**: Total open posts, active applications, resolved enquiries.
- **Financials**: Total registration payments collected this month vs last month.
- **Conversion Rates**: Percentage of applications that reach DC, GC, and Approval.
- **Demographics**: Top requested subjects and locations (driving marketing decisions).

## 3. The Unified Calendar

Instead of checking multiple pages to see what needs to be done today, the Unified Calendar aggregates:
- **DC (Demo Calls)**: Scheduled with providers.
- **GC (Guardian Confirmations)**: Follow-ups with consumers.
- **Enquiry Follow-ups**: Reminders to call a lead back.
- **Custom Todos**: Manually created tasks for admins.

This is powered by the `CalendarEvent` model. Whenever an admin sets a `dcDate` on an application, a Mongoose hook automatically upserts a `CalendarEvent`.

## 4. The Ledger System (Sheets Sync)

AOTF replaces standalone spreadsheets while still embracing them as a secondary viewing tool.

The `PostLedger` and `EnquiryLedger` collections store enriched, flat versions of complex data. Whenever a Post or Enquiry is updated, the respective ledger is updated, and an API call is made to a **Google Sheet**.

This gives the admin team the best of both worlds:
- A robust, relational database (MongoDB) ensuring data integrity.
- A familiar, easy-to-read spreadsheet (Google Sheets) for quick overviews or sharing with external stakeholders (without giving them admin access).

## 5. Invoicing

Admins can generate PDF-ready invoices directly from the `PostLedger` view.
- Invoices are versioned. If a mistake is made, a new version is created rather than overwriting the old one, preserving the audit trail.
- Payment tracking handles partial payments (e.g., consumer pays half upfront, half later).

## 6. Audit & Activity Logs

Every action is tracked.
- **Activity Log**: Shows who updated an enquiry, who created a post, etc. Crucial for team coordination.
- **Audit Log**: High-security tracking. Shows who exported data, who changed an admin's role, or who issued a refund.

## 7. Users Management

The **Users** page (`/admin/users`) lists provider accounts from MongoDB with automatic Clerk sync on load.

- Teacher and candidate tabs with status filters
- Block, unblock, and soft-delete via `/api/admin/app-users`
- Create-user FAB for manual provisioning
- Post-migration visibility after [legacy import](/docs/how-to/migrate-legacy-users)

See [Manage Users in Admin](/docs/how-to/admin-manage-users) and [Admin App Users API](/docs/reference/api/app-users).
