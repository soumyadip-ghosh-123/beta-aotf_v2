# Enquiries Deep Dive

Enquiries represent the very top of the AOTF funnel. They are raw leads—a consumer expressing a need for a tutor or freelancer.

## Sources of Enquiries

Enquiries come into the platform through multiple channels:

1. **Public Website Form**: Consumers fill out a form at `aotf.in/contact`. This creates a `new` Enquiry automatically.
2. **Direct Call / WhatsApp**: A consumer texts the AOTF business number. An admin manually creates the Enquiry in the dashboard.
3. **Referrals**: Handled the same as calls/WhatsApp.

## The CRM Pipeline

AOTF uses a lightweight CRM pipeline to track these leads.

1. **New**: The lead has arrived. No one has touched it yet.
2. **In Progress**: An admin claims the lead and begins working it.
3. **Contacted**: The admin has successfully spoken to the consumer to gather details.
4. **Unreachable**: The consumer is not picking up or replying.
5. **Resolved**: The need has been met (a post was created and a teacher was found).
6. **Closed**: The lead was spam, a duplicate, or the consumer changed their mind.

Every time an admin changes the status, they must provide a **Note** (e.g., "Called them, they want a female tutor for ICSE board"). This creates an `EnqStatus` audit log entry, so the team has a full history of the interaction.

## Google Sheets Sync (EnquiryLedger)

Because admins are accustomed to viewing leads in a spreadsheet, every time an Enquiry is updated, it syncs to the "Enquiries" tab of a shared Google Sheet.

The `EnquiryLedger` collection holds extra fields specifically for this sync (like `callAttempts`, `assignedTeacherName`). The Mongoose `post('save')` hook fires `upsertEnquiryLedger()`, which handles the Sheets API call asynchronously.

## Converting Enquiries

When a requirement is fully understood, the admin clicks "Create Post" (or "Create Job") from the Enquiry view.

This does three things:
1. Opens the Post creation form, pre-filled with the consumer's details.
2. Links the `enquiryId` to the newly created Post.
3. Marks the EnquiryLedger as `convertedToPost: true`.

This linkage allows AOTF to track conversion rates (how many leads turn into actual assignments) and ensures that admins don't lose the original context of the consumer's request.
