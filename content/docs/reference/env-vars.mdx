# Environment Variables

All environment variables live in `.env.local` for development and in **Vercel â†’ Settings â†’ Environment Variables** for production. Never commit `.env.local` to version control.

## MongoDB

| Variable | Required | Description |
|---|---|---|
| `MONGODB_URI` | âś… | New app database connection string |
| `MONGODB_URI_LEGACY` | Migration only | Legacy production cluster (`academy-of-tutorials-freelancers`). Used by [migration scripts](/docs/how-to/seed-users). Not needed at runtime after migration. |

```bash

MONGODB_URI_LEGACY=mongodb+srv://...@legacy-cluster/academy-of-tutorials-freelancers
```

## Google Sheets

| Variable | Required | Description |
|---|---|---|
| `GOOGLE_SERVICE_ACCOUNT_EMAIL` | âś… | Service account email |
| `GOOGLE_SERVICE_ACCOUNT_PRIVATE_KEY` | âś… | RSA private key (with literal `\n`) |
| `GOOGLE_SHEETS_SPREADSHEET_ID` | âś… | Spreadsheet ID from the URL |

```bash

GOOGLE_SERVICE_ACCOUNT_PRIVATE_KEY="-----BEGIN RSA PRIVATE KEY-----\nMII...\n-----END RSA PRIVATE KEY-----\n"
GOOGLE_SHEETS_SPREADSHEET_ID=1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms
```

> **Private key formatting**: The key must have literal `\n` characters. Vercel stores them correctly. In `.env.local`, wrap in double quotes.

## Sentry (Monitoring)

| Variable | Required | Description |
|---|---|---|
| `NEXT_PUBLIC_SENTRY_DSN` | Optional | Runtime DSN â€” enables error tracking in browser and server |
| `SENTRY_AUTH_TOKEN` | Optional | Build-time only â€” uploads source maps during CI/Vercel build |
| `SENTRY_ORG` | Optional | Build-time only â€” Sentry org slug (must match `next.config.js`) |
| `SENTRY_PROJECT` | Optional | Build-time only â€” Sentry project slug (must match `next.config.js`) |

```bash
NEXT_PUBLIC_SENTRY_DSN=https://...@sentry.io/...
SENTRY_AUTH_TOKEN=sntrys_...
SENTRY_ORG=aotf
SENTRY_PROJECT=javascript-nextjs
```

Only `NEXT_PUBLIC_SENTRY_DSN` is needed at runtime. The three `SENTRY_*` variables are optional but recommended for readable stack traces in production. See [Monitoring with Sentry](/docs/explanations/monitoring-sentry) for the full setup and [Triage Sentry Errors](/docs/how-to/triage-sentry-errors) for incident response.

## Quick Validation

Run the health check to verify env vars are set correctly:

```bash
# Start dev server
pnpm dev

# In another terminal â€” should return { "status": "ok", "db": "connected" }
curl http://localhost:3000/api/health | jq .
```

If `db: "disconnected"`, check `MONGODB_URI`. If the server fails to start, check for missing `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` or `CLERK_SECRET_KEY`.
