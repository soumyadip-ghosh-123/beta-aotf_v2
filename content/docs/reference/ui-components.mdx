# UI Components & Dependencies

AOTF uses a rich UI stack built on top of Next.js with a combination of component libraries, animation tools, and utility packages.

## Core Component Libraries

### HeroUI (formerly NextUI) v2

The primary component library. AOTF uses individual HeroUI packages for tree-shaking:

```json
"@heroui/button": "^2.2.14",
"@heroui/calendar": "^2.2.10",
"@heroui/card": "^2.2.10",
"@heroui/checkbox": "^2.3.11",
"@heroui/chip": "^2.2.10",
"@heroui/date-picker": "^2.3.10",
"@heroui/dropdown": "^2.3.11",
"@heroui/form": "^2.1.10",
"@heroui/image": "^2.2.9",
"@heroui/input": "^2.3.11",
"@heroui/modal": "^2.2.11",
"@heroui/popover": "^2.3.11",
"@heroui/progress": "^2.2.10",
"@heroui/radio": "^2.3.11",
"@heroui/select": "^2.3.11",
"@heroui/spinner": "^2.2.7",

```

HeroUI is a Tailwind-based component system. All HeroUI components use the global `tailwind.config.ts` for theme customization.

### Aceternity UI

For premium animated components (hero sections, spotlight effects, floating elements):

```json
"@aceternity/ui": "^1.x"
```

Custom Aceternity components are in `components/aceternity/`.

### React Bits

For additional interactive UI elements and micro-interactions:

```json
"react-bits": "^x.x"
```

### Radix UI

Low-level accessible primitives used directly in custom components:

```json
"@radix-ui/react-dialog": "^x",
"@radix-ui/react-dropdown-menu": "^x",
"@radix-ui/react-popover": "^x",
"@radix-ui/react-select": "^x",
"@radix-ui/react-slot": "^x",
"@radix-ui/react-tooltip": "^x"
```

## Rich Text / Content Editing

Used for the rich text editor in admin forms (application notes, feedback, admin messages):

```json
"@tiptap/core": "^2.x",
"@tiptap/react": "^2.x",
"@tiptap/starter-kit": "^2.x",
"@tiptap/extension-link": "^2.x",
"@tiptap/extension-placeholder": "^2.x"
```

## Animation Libraries

### Framer Motion

Used for page transitions, modal animations, and component enter/exit animations:

```json
"framer-motion": "^12.x"
```

### GSAP (GreenSock Animation Platform)

Used for complex scroll-triggered animations on the landing page and admin dashboard:

```json
"gsap": "^3.x"
```

### Lottie React

Used for micro-animation JSON files (loading states, success indicators):

```json
"lottie-react": "^2.x"
```

## Data Fetching

AOTF uses **SWR** for client-side data fetching with cache invalidation:

```json
"swr": "^2.x"
```

Server-side data is fetched in Server Components using `fetch` with Next.js caching directives.

## Admin-Specific UI

The admin panel (`/admin/*`) uses a custom layout with:

- `react-big-calendar` — Full admin calendar view with drag-and-drop (OSS, credited in the calendar view)
- `recharts` — Analytics dashboard charts
- `@tanstack/react-table` — Server-side paginated data tables
- `Tiptap` — Rich text notes on applications/enquiries

## Custom Component Directory

```
components/
├── aceternity/      # Aceternity UI customized components
├── admin/           # Admin-specific components
├── docs/            # Documentation components (DocToolbar, etc.)
├── mdx/             # MDX custom element renderers
├── providers/       # Context providers (HeroUI, theme, etc.)
├── shared/          # Shared components used across both admin and user UIs
└── ui/              # Base UI components (Button wrappers, etc.)
```
