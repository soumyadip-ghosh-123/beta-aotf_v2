# Enquiry, EnquiryLedger & EnqStatus Models

## Enquiry

**Collection**: `enquiries`  
**Source**: [`lib/models/Enquiry.ts`](file:///d:/aotf_v2/lib/models/Enquiry.ts)

The primary record for a consumer requirement. Created when a consumer submits the public enquiry form or when an admin logs a phone/WhatsApp enquiry.

### Schema

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `_id` | `ObjectId` | Auto | — | |
| `enquiryId` | `string` | ✅ | — | Human-readable ID (e.g., `ENQ-001`, unique) |
| `name` | `string` | ✅ | — | Consumer name |
| `phoneNumber` | `string` | ✅ | — | Consumer phone |
| `query` | `string` | ✅ | — | Requirement description |
| `currentStatus` | `enum` | — | `"new"` | `new`, `in_progress`, `contacted`, `unreachable`, `resolved`, `closed` |
| `source` | `string` | — | `null` | How the enquiry was received |
| `lastActionNote` | `string` | — | `null` | Most recent admin note |
| `handledByAdminClerkId` | `string` | — | `null` | Assigned admin |
| `linkedPostId` | `string` | — | `null` | Post created from this enquiry |
| `linkedJobId` | `string` | — | `null` | Job created from this enquiry |
| `createdAt` | `Date` | Auto | — | |
| `updatedAt` | `Date` | Auto | — | |

### Indexes

```ts
{ enquiryId: 1 }           // unique
{ currentStatus: 1, createdAt: -1 }
{ phoneNumber: 1 }
{ handledByAdminClerkId: 1 }
```

### Calendar & Sheets Hooks

Enquiry has Mongoose `post` hooks:

```ts
EnquirySchema.post("save", function (doc) {
  // Sync to EnquiryLedger → Google Sheets
  void upsertEnquiryLedger(doc.toObject());
  // Create/update CalendarEvent if followUpDate is set
  if (doc.followUpDate) {
    void upsertCalendarEvent(mapEnquiryToEvent(doc));
  }
});
```

## EnquiryLedger

**Collection**: `enquiry_ledgers`

A CRM-enriched view of each enquiry, optimized for Google Sheets sync. Contains all `Enquiry` fields plus additional tracking fields added by admins over time.

### Additional Fields (beyond Enquiry)

| Field | Type | Description |
|---|---|---|
| `callAttempts` | `number` | Number of call attempts made |
| `followUpDate` | `Date` | Next scheduled follow-up |
| `convertedToPost` | `boolean` | Whether a Post was created |
| `convertedToJob` | `boolean` | Whether a Job was created |
| `assignedTeacherName` | `string` | Teacher name if matched |
| `sheetRowIndex` | `number` | Row number in "Enquiries" Google Sheet tab |
| `lastSyncedAt` | `Date` | Last Google Sheets sync timestamp |
| `statusHistory` | `array` | Embedded status change records |

### Google Sheets Sync

Every `EnquiryLedger` upsert triggers a write to the **"Enquiries"** tab in the AOTF Google Sheet:

```ts
// lib/services/enquiryLedger.service.ts
await sheets.spreadsheets.values.update({
  spreadsheetId: process.env.GOOGLE_SHEETS_SPREADSHEET_ID,
  range: `Enquiries!A${rowIndex}:Z${rowIndex}`,
  valueInputOption: "RAW",
  requestBody: { values: [mapLedgerToRow(ledger)] },
});
```

## EnqStatus

**Collection**: `enq_statuses`

Complete audit trail of status changes for every enquiry.

### Schema

| Field | Type | Description |
|---|---|---|
| `enquiryId` | `ObjectId` | Ref: `Enquiry._id` |
| `fromStatus` | `enum` | Previous status |
| `toStatus` | `enum` | New status |
| `changedByClerkId` | `string` | Admin Clerk ID |
| `changedAt` | `Date` | Timestamp |
| `note` | `string` | Admin note for this change |

### Index

```ts
{ enquiryId: 1, changedAt: -1 }
```
