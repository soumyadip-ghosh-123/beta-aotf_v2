# CalendarEvent & TodoEvent Models

## CalendarEvent

**Collection**: `calendar_events`  
**Source**: `lib/models/CalendarEvent.ts`

The unified calendar event model. All date-based events from every source (applications, enquiries, todos, feedback) are funnelled into this collection via write-through Mongoose hooks. This powers the admin calendar UI.

### Schema

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `_id` | `ObjectId` | Auto | — | |
| `eventKey` | `string` | ✅ | — | Unique business key (see below) |
| `title` | `string` | ✅ | — | Human-readable event title |
| `description` | `string` | — | `null` | Optional description |
| `startAt` | `Date` | ✅ | — | Event start time |
| `endAt` | `Date` | — | `null` | Event end time |
| `category` | `enum` | ✅ | — | `tuition`, `job`, `enquiry`, `feedback`, `reminder` |
| `refId` | `ObjectId` | — | `null` | Ref to source document |

| `handledByAdminId` | `ObjectId` | — | `null` | Assigned admin |

### eventKey Design

The `eventKey` is a **stable, deterministic business key** that enables upsert-style calendar syncs without creating duplicates:

```ts
// Pattern: <source>:<sourceId>:<eventType>
"application:app_001:dc"         // Demo Call for application app_001
"application:app_001:gc"         // Guardian Confirmation for app_001
"enquiry:ENQ-001:followup"       // Follow-up for enquiry ENQ-001
"todo:todoId123:reminder"        // Admin todo event
"feedback:fbk_001:review"        // Feedback review event
```

The `upsertCalendarEvent` service uses `eventKey` to find-and-update or insert:

```ts
await CalendarEvent.findOneAndUpdate(
  { eventKey: event.eventKey },
  { $set: event, $inc: { syncVersion: 1 } },
  { upsert: true, new: true }
);
```

### Write-Through Sources

| Source Model | Trigger | Category |
|---|---|---|
| `Application` | `post("findOneAndUpdate")` when dcDate set | `tuition` |
| `Enquiry` | `post("save")` when followUpDate set | `enquiry` |
| `TodoEvent` | `post("save")` + `post("findOneAndUpdate")` | varies |
| `Feedback` | `post("save")` | `feedback` |

## TodoEvent

**Collection**: `todo_events`  
**Source**: [`lib/models/TodoEvent.ts`](file:///d:/aotf_v2/lib/models/TodoEvent.ts)

Admin task and reminder model. Tied to CRM entities (posts, enquiries, jobs, feedback).

### Schema

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `category` | `enum` | ✅ | — | `tuition`, `enquiry`, `job`, `feedback` |
| `status` | `enum` | ✅ | — | Varies by category (see below) |
| `title` | `string` | ✅ | — | Max 200 chars |
| `note` | `string` | — | `null` | Optional description |
| `dueAt` | `Date` | ✅ | — | When the todo is due |
| `refId` | `ObjectId` | — | `null` | Source document `_id` |
| `refLabel` | `string` | — | `null` | Human-readable source ID |
| `handledByAdminId` | `ObjectId` | — | `null` | Assigned admin |
| `handledByAdminName` | `string` | — | `null` | Name snapshot |
| `isDone` | `boolean` | — | `false` | Completion flag |

### Status Values by Category

| Category | Valid Statuses |
|---|---|
| `tuition` | `demo_reminder`, `demo_confirmation`, `guardian_confirmed`, `completed` |
| `enquiry` | `pending`, `resolved` |
| `job` | `pending`, `sent_to_company` |
| `feedback` | `open`, `under_review`, `action_taken`, `resolved` |

### Indexes

```ts
{ dueAt: 1, isDone: 1 }
{ category: 1, dueAt: -1 }
{ refId: 1 }
{ handledByAdminId: 1, dueAt: 1 }
```

### Calendar Hooks

```ts
TodoEventSchema.post("save", (doc) => {
  void upsertCalendarEvent(mapTodo(doc.toObject()));
});

TodoEventSchema.post("findOneAndDelete", (doc) => {
  if (!doc) return;
  void deleteCalendarEvent(todoEventKey(doc._id.toString()));
});
```

When a TodoEvent is deleted, the corresponding `CalendarEvent` is also removed.
