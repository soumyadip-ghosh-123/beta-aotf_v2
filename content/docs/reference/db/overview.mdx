# Database Overview

AOTF uses **MongoDB Atlas M0** (free tier) as the primary data store, accessed via **Mongoose** ODM.

## Connection Details

| Setting | Value |
|---|---|
| Provider | MongoDB Atlas |
| Tier | M0 (Free) |
| Region | Mumbai (ap-south-1) |
| Storage | 512 MB max |
| Connection | Shared cluster |
| Auto-pause | After 60 min inactivity |

### Connection String Format

```
mongodb+srv://<user>:<password>@<cluster>.mongodb.net/<database>?retryWrites=true&w=majority
```

The connection string is stored in `MONGODB_URI` and must never be committed to version control.

## Connection Logic (`lib/db.ts`)

The `dbConnect()` function implements a **singleton connection cache** to avoid creating multiple connections in serverless environments:

```ts
// lib/db.ts (simplified)
const RETRY_COUNT = 3;
const BASE_DELAY_MS = 2000; // doubles each retry: 2s → 4s → 8s

let cachedConnection = global._mongooseConnection;

export default async function dbConnect() {
  if (cachedConnection?.conn) return cachedConnection.conn;

  for (let attempt = 1; attempt <= RETRY_COUNT; attempt++) {
    try {
      const conn = await mongoose.connect(process.env.MONGODB_URI!, {
        serverSelectionTimeoutMS: 15_000,
        socketTimeoutMS: 45_000,
      });
      cachedConnection = { conn };
      return conn;
    } catch (err) {
      if (attempt === RETRY_COUNT) throw err;
      await sleep(BASE_DELAY_MS * 2 ** (attempt - 1)); // exponential back-off
    }
  }
}
```

See [Cold Starts & Free Tier](/docs/explanations/cold-start) for full details on the retry and warm-up strategy.

## Mongoose Conventions

- **All models** use `timestamps: true` unless explicitly disabled
- **Soft deletes** — most user-facing data is never hard deleted; `status: "deleted"` is used instead
- **`lean()` queries** — all read-only queries use `.lean()` for performance (returns plain JS objects, not Mongoose documents)
- **Indexes** — every model declares explicit compound indexes for its common query patterns
- **Singleton pattern** — `models.ModelName || mongoose.model(...)` prevents duplicate model registration in serverless environments

## NoSQL Injection Protection

Mongoose automatically sanitizes user input by default. For dynamic filter building, `sanitizeFilter` is used:

```ts
import { sanitizeFilter } from "mongoose";

const filter = sanitizeFilter({ username: req.query.username });
// Strips $operators that could enable injection attacks
```
