# Admin Models

## Admin

**Collection**: `admins`  
**Source**: `lib/models/Admin.ts`

The primary model for admin accounts. Mirrors the corresponding Clerk user with an additional permissions object.

### Schema

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `_id` | `ObjectId` | Auto | — | |
| `clerkId` | `string` | ✅ | — | Clerk user ID (unique) |
| `username` | `string` | ✅ | — | Lowercase admin username |
| `email` | `string` | ✅ | — | Admin email address |
| `name` | `string` | ✅ | — | Full display name |
| `role` | `enum` | ✅ | — | `super_admin`, `admin`, `support_admin`, `crm` |
| `permissions` | `object` | ✅ | — | 23 boolean permission flags (see below) |
| `isActive` | `boolean` | — | `true` | Account active flag |
| `isLocked` | `boolean` | — | `false` | Locked after 5 failed logins |
| `requirePasswordChange` | `boolean` | — | `false` | Force password reset on next login |
| `createdBy` | `ObjectId` | — | `null` | Ref: `Admin._id` who created this admin |
| `terminatedAt` | `Date` | — | `null` | When isActive was set to false |
| `createdAt` | `Date` | Auto | — | |
| `updatedAt` | `Date` | Auto | — | |

### Permissions Object (23 fields)

```ts
{
  canManageUsers: boolean;
  canBlockUsers: boolean;
  canManagePosts: boolean;
  canManageJobs: boolean;
  canCreateTuitionPosts: boolean;
  canCreateJobPosts: boolean;
  canEditPosts: boolean;
  canDeletePosts: boolean;
  canHandleEnquiries: boolean;
  canHandleFeedbacks: boolean;
  canUpdateEnquiryStatus: boolean;
  canCallApplicants: boolean;
  canProcessRefunds: boolean;
  canViewPayments: boolean;
  canViewAnalytics: boolean;
  canExportData: boolean;
  canManageAdmins: boolean;
  canCreateAdmins: boolean;
  canEditAdmins: boolean;
  canDeactivateAdmins: boolean;
  canResetAdminPasswords: boolean;
  canTerminateAdmins: boolean;
  canViewAuditLogs: boolean;
}
```

### Static Method: `getDefaultPermissions(role)`

Returns the default permission set for a given role:

```ts
const permissions = Admin.getDefaultPermissions("super_admin");
// → all 23 permissions set to true
```

See the [full permissions matrix](/docs/reference/admin/permissions) for default values per role.

## AdminUser

**Collection**: `admin_users`  
**Source**: `lib/models/admin/AdminUser.ts`

Tracks admin account lifecycle states beyond what the `Admin` model captures — specifically for the admin join/status workflow.

### Schema

| Field | Type | Description |
|---|---|---|
| `adminId` | `ObjectId` | Ref: `Admin._id` |
| `clerkId` | `string` | Clerk user ID |
| `accountStatus` | `enum` | `pending_setup`, `active`, `suspended`, `terminated` |
| `inviteToken` | `string` | Token from AdminInvite (cleared after use) |
| `joinedAt` | `Date` | When account setup was completed |
| `suspendedAt` | `Date` | When account was suspended |
| `suspendedBy` | `ObjectId` | Admin who suspended |
| `terminatedAt` | `Date` | When account was terminated |
| `terminatedBy` | `ObjectId` | Admin who terminated |

## AdminInvite

**Collection**: `admin_invites`

Stores invite tokens for new admin onboarding via `POST /api/admin/join`.

### Schema

| Field | Type | Description |
|---|---|---|
| `email` | `string` | Invited admin email |
| `role` | `enum` | Pre-assigned role |
| `name` | `string` | Pre-assigned display name |
| `token` | `string` | Secure random token (unique) |
| `createdBy` | `ObjectId` | Admin who sent the invite |
| `usedAt` | `Date` | When the invite was redeemed |
| `expiresAt` | `Date` | TTL — invite expires after 7 days |
| `isUsed` | `boolean` | Whether the token has been redeemed |

### TTL Index

```ts
{ expiresAt: 1 }, { expireAfterSeconds: 0 }
```

Expired invites are automatically deleted by MongoDB.

## AdminRole (Legacy)

**Collection**: `admin_roles`

A legacy model used for role configuration. Currently the role definitions are embedded in the `create-admin.ts` script and `Admin.getDefaultPermissions()`. This collection is no longer actively written to but may contain historical records.

### Schema

| Field | Type | Description |
|---|---|---|
| `name` | `string` | Role name |
| `permissions` | `object` | Permission flags |
| `description` | `string` | Human-readable role description |
