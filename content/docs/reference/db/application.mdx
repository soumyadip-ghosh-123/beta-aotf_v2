# Application & ApplicationEvent Models

## Application

**Collection**: `applications`  
**Source**: [`lib/models/Application.ts`](file:///d:/aotf_v2/lib/models/Application.ts)

Links a provider to a Post or Job listing. Stores the full status lifecycle and scheduling metadata for the DC/GC flow.

### Schema

| Field | Type | Required | Default | Description |
|---|---|---|---|---|

| `applicantClerkId` | `string` | ✅ | — | Provider's Clerk ID |
| `applicantUsername` | `string` | ✅ | — | Snapshot of username at apply time |
| `applicantId` | `ObjectId` | — | `null` | Ref: `User._id` |

### Indexes

```ts
{ applicantClerkId: 1, postId: 1 }   // compound, unique per applicant+post
{ applicantClerkId: 1, jobId: 1 }    // compound, unique per applicant+job
{ postId: 1, status: 1 }
{ jobId: 1, status: 1 }
{ status: 1, appliedAt: -1 }
```

### Validation Rules

- Either `postId` OR `jobId` must be set (not both, not neither)
- Providers cannot apply twice to the same post/job (compound unique index)
- Cannot apply to a `closed`, `cancelled`, or `matched` post/job

### Calendar Hooks

`Application` has Mongoose `post` hooks that sync to `CalendarEvent`:

```ts
// lib/models/Application.ts
ApplicationSchema.post("findOneAndUpdate", function (doc) {
  if (doc?.dcDate) {
    void upsertCalendarEvent(mapApplicationToDCEvent(doc));
  }
});
```

When `dcDate` is set, a `CalendarEvent` is created/updated for that date under the `tuition` category.

## ApplicationEvent

**Collection**: `application_events`  
**Source**: `lib/models/ApplicationEvent.ts`

Complete audit trail of every status change for every application.

### Schema

| Field | Type | Description |
|---|---|---|
| `applicationId` | `ObjectId` | Ref: `Application._id` |
| `fromStatus` | `enum` | Previous status |
| `toStatus` | `enum` | New status |
| `changedByClerkId` | `string` | Admin (or system) Clerk ID |
| `changedAt` | `Date` | Timestamp |
| `note` | `string` | Optional admin note |
| `metadata` | `object` | Any extra context (e.g., dcDate, declineReason) |

### Example

```json
{
  "applicationId": "...",
  "fromStatus": "applied",
  "toStatus": "DC",
  "changedByClerkId": "user_admin123",
  "changedAt": "2026-01-10T12:00:00.000Z",
  "note": "Scheduled demo for Jan 15",
  "metadata": { "dcDate": "2026-01-15T15:00:00.000Z" }
}
```
