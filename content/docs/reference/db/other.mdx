# Other Models

## Invoice

**Collection**: `invoices`  
**Source**: [`lib/models/Invoice.ts`](file:///d:/aotf_v2/lib/models/Invoice.ts)

Consumer-facing invoices generated by admins. Supports **versioning** and **partial payments**.

### Key Fields

| Field | Type | Description |
|---|---|---|
| `invoiceId` | `string` | Human-readable ID (e.g., `INV-001`) |
| `version` | `number` | Version number (starts at 1, increments on edit) |
| `isLatest` | `boolean` | Only the latest version is `true` |
| `postId` | `string` | Linked post ID |
| `guardianName` | `string` | Consumer name |
| `guardianPhone` | `string` | Consumer phone |
| `amount` | `number` | Invoice total in INR |
| `paidAmount` | `number` | Amount paid so far |
| `paymentStatus` | `enum` | `unpaid`, `partial`, `paid` |
| `dueDate` | `Date` | Payment due date |
| `items` | `array` | Line items (description, quantity, rate, amount) |
| `notes` | `string` | Admin notes on invoice |
| `generatedByAdminClerkId` | `string` | Admin who created the invoice |

### Versioning Pattern

When an invoice is edited:
1. The existing document's `isLatest` is set to `false`
2. A new document is created with `version: previous + 1` and `isLatest: true`

This preserves a full history of all invoice versions.

## AdminActivityLog

**Collection**: `admin_activity_logs`  
**Source**: [`lib/models/admin/AdminActivityLog.ts`](file:///d:/aotf_v2/lib/models/admin/AdminActivityLog.ts)

Day-to-day CRM operations log. More granular than `AuditLog` — captures routine admin work.

### 7 Modules

| Module | Actions Captured |
|---|---|
| `CRM` | Enquiry status updates, follow-ups |
| `FRM` | Post/job creation, application moves |
| `COMMS` | Notification sends, call logs |
| `CALENDAR` | Calendar event creation/edits |
| `LEDGER` | PostLedger updates, payment recording |
| `ADMIN_MGMT` | Admin account changes |
| `USER_MGMT` | Provider account updates |

### Key Fields

| Field | Type | Description |
|---|---|---|
| `module` | `enum` | One of the 7 modules |
| `action` | `string` | Specific action performed |
| `adminClerkId` | `string` | Admin who performed |
| `adminName` | `string` | Admin name snapshot |
| `entityId` | `string` | ID of affected entity |
| `entityType` | `string` | Type of affected entity |
| `diff.before` | `object` | State before the change |
| `diff.after` | `object` | State after the change |
| `ipAddress` | `string` | Admin IP |
| `browser` | `string` | Browser info |
| `os` | `string` | OS info |
| `location` | `string` | Approximate geo location |
| `performedAt` | `Date` | Timestamp |

The `diff.before/after` fields enable full change auditing — you can see exactly what was changed on any record.

## WebhookEvent

**Collection**: `webhook_events`

Idempotency deduplication records for webhook deliveries.

| Field | Type | Description |
|---|---|---|
| `provider` | `enum` | `clerk`, `razorpay` |
| `event` | `string` | Event type (e.g., `user.created`) |
| `entityId` | `string` | Entity from the event payload |
| `payload` | `object` | Full event payload |
| `signature` | `string` | Svix/Razorpay signature |
| `processed` | `boolean` | Whether processing completed |
| `processedAt` | `Date` | When processing completed |

Unique index on `{ provider, event, entityId }` ensures each event is processed only once.
