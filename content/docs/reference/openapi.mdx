# OpenAPI Explorer

Explore all AOTF API endpoints interactively. The specification is served live from [`/api/openapi.json`](https://aotf.in/api/openapi.json) and covers all routes, request schemas, response shapes, and authentication requirements.

## Using the Explorer

- **Expand** any endpoint to see request/response schemas
- **Try it out** to send test requests directly from the browser (requires a valid Clerk JWT)
- **Download** the spec as JSON for use in Postman, Insomnia, or code generation tools

For Postman, use the pre-built collection at [`/docs/AOTF.postman_collection.json`](/docs/AOTF.postman_collection.json).

## Authentication in the Explorer

To test authenticated endpoints:
1. Get your Clerk JWT from the browser console while logged in: `await window.Clerk.session.getToken()`
2. Click **Authorize** at the top of the explorer
3. Paste the token in the **ClerkBearer** field

## Spec Source

The spec is hand-authored in [`app/api/openapi.json/route.ts`](file:///d:/aotf_v2/app/api/openapi.json/route.ts) and follows the **OpenAPI 3.1.0** standard.

When you add new API routes, update this file to keep the spec current.
