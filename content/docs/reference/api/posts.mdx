# Posts API

Tuition posts represent consumer requirements for tutors. They are created by admins and publicly listed for providers to browse and apply.

## Post Statuses

| Status | Meaning |
|---|---|
| `open` | Accepting applications |
| `matched` | A teacher has been approved |
| `closed` | Assignment complete |
| `cancelled` | Post cancelled by admin/consumer |
| `hold` | Temporarily paused |

## List Posts

```http
GET /api/v1/posts
```

Public endpoint. No authentication required.

**Query Parameters**:

| Param | Type | Description |
|---|---|---|
| `page` | `number` | Page (default: 1) |
| `limit` | `number` | Per page (default: 20) |
| `status` | `string` | Filter: `open`, `matched`, `closed`, `cancelled`, `hold` |
| `classType` | `string` | Filter: `online`, `offline`, `both` |
| `search` | `string` | Full-text search on location, subjects |

**Response 200**:

## Get Single Post

```http
GET /api/v1/posts/:postId
```

Public endpoint. `postId` is the human-readable ID (e.g., `POST-001`), not the MongoDB `_id`.

**Response 404** if not found.

## Create Post

```http
POST /api/v1/posts
Authorization: Bearer <admin_clerk_jwt>
Content-Type: application/json
Origin: https://aotf.in
```

**Permission required**: `canCreateTuitionPosts`

**Request Body**:
```json
{
  "guardianName": "Mrs. Sharma",
  "guardianPhone": "9876543210",
  "enquiryId": "6843f2a1b4e5c9d0e8f12345",
  "source": "WhatsApp",
  "students": [
    {
      "className": "Class 9",
      "board": "CBSE",
      "subjects": ["Mathematics", "Science"]
    }
  ],
  "classType": "offline",
  "frequencyPerWeek": 3,
  "preferredDays": ["Mon", "Wed", "Fri"],
  "preferredTime": "5–7 PM",
  "location": "Salt Lake, Kolkata",
  "monthlyBudget": 3000,
  "notes": "Prefers female teacher"
}
```

**Required fields**: `guardianName`, `guardianPhone`, `source`, `students` (min 1), `classType`, `frequencyPerWeek`, `location`, `monthlyBudget`

**Response 201**: Created post document with auto-generated `postId`.

**Side effects**:
- Creates a `PostLedger` document
- Syncs to "Tuitions" Google Sheet tab
- Creates a `CalendarEvent` if a `dueDate` is set

## Update Post

```http
PATCH /api/v1/posts/:postId
Authorization: Bearer <admin_clerk_jwt>
Content-Type: application/json
Origin: https://aotf.in
```

**Permission required**: `canEditPosts`

All fields are optional. Only provided fields are updated.

**Status transitions**:
- `open` → `matched` (when an application is approved)
- `matched` → `closed` (when assignment is complete)
- Any status → `cancelled` or `hold` (admin discretion)

**Response 200**: Updated post document.

**Side effects**: PostLedger and Google Sheet are updated automatically.

## Delete Post

```http
DELETE /api/v1/posts/:postId
Authorization: Bearer <admin_clerk_jwt>
Origin: https://aotf.in
```

**Permission required**: `canDeletePosts`

**Hard delete** — the post is permanently removed from MongoDB, PostLedger, and Google Sheet.

**Response 200**: `{ "success": true }`

## Post Class Types

| Value | Meaning |
|---|---|
| `online` | Remote/virtual sessions only |
| `offline` | In-person sessions only |
| `both` | Either format acceptable |

## Student Sub-document

Each post has one or more `students` entries:

```ts
{
  className: string;          // e.g., "Class 9", "Class 12"
  board: string;              // e.g., "CBSE", "ICSE", "WB"
  subjects: string[];         // e.g., ["Mathematics", "Physics"]
  subjectsNormalized: string[]; // lowercase, auto-generated for search
}
```

The `subjectsNormalized` field is auto-generated by the API before save — do not send it in the request.
