# Enquiry API

Enquiries are the entry point for consumers reaching out to AOTF. The public submission form is rate-limited and requires no auth. Admin endpoints for reading and updating enquiries require authentication.

## Enquiry Statuses

| Status | Meaning |
|---|---|
| `new` | Just submitted, no action taken |
| `in_progress` | Admin is actively handling |
| `contacted` | Admin or provider has contacted the consumer |
| `unreachable` | Consumer is not responding |
| `resolved` | Requirement fulfilled |
| `closed` | Closed without fulfillment |

## Submit Enquiry (Public)

```http
POST /api/v1/enquiry
Content-Type: application/json
```

**No authentication required.** Rate-limited to **5 requests per IP per minute**.

**Request Body**:
```json
{
  "name": "Mrs. Sharma",
  "phoneNumber": "9876543210",
  "query": "Need a Math tutor for Class 9 CBSE in Salt Lake area"
}
```

**Required fields**: `name`, `phoneNumber`, `query`

**Response 201**:
```json
{
  "success": true,
  "enquiryId": "ENQ-001",
  "message": "Your enquiry has been received. We will contact you shortly."
}
```

**Response 429** (rate limited):
```json
{
  "error": "Too many requests. Please try again later.",
  "retryAfter": 45
}
```

**Side effects**:
- Creates an `Enquiry` document with `currentStatus: "new"`
- Creates an `EnquiryLedger` document
- Syncs to "Enquiries" Google Sheet tab
- Triggers notification to admin team (email via Resend)

## List Enquiries (Admin)

```http
GET /api/v1/enquiry
Authorization: Bearer <admin_clerk_jwt>
```

**Permission required**: `canHandleEnquiries`

**Query Parameters**:

| Param | Type | Description |
|---|---|---|
| `status` | `string` | Filter by status |
| `page` | `number` | Page (default: 1) |
| `limit` | `number` | Per page (default: 20) |
| `search` | `string` | Search by name or phone |
| `from` | `string` | Date range start (ISO-8601) |
| `to` | `string` | Date range end (ISO-8601) |

**Response 200**:
```json
{
  "enquiries": [
    {
      "_id": "...",
      "enquiryId": "ENQ-001",
      "name": "Mrs. Sharma",
      "phoneNumber": "9876543210",
      "query": "Need a Math tutor...",
      "currentStatus": "new",
      "source": "website_form",
      "createdAt": "2026-01-01T00:00:00.000Z",
      "lastActionNote": null
    }
  ],
  "total": 87,
  "page": 1
}
```

## Get Single Enquiry (Admin)

```http
GET /api/v1/enquiry/:enquiryId
Authorization: Bearer <admin_clerk_jwt>
```

**Permission required**: `canHandleEnquiries`

Returns full enquiry document including status history from the `EnqStatus` collection.

**Response 200**:
```json
{
  "enquiry": { ... },
  "statusHistory": [
    {
      "status": "new",
      "changedAt": "2026-01-01T00:00:00.000Z",
      "changedByClerkId": null
    },
    {
      "status": "in_progress",
      "changedAt": "2026-01-01T02:00:00.000Z",
      "changedByClerkId": "user_admin..."
    }
  ],
  "linkedPost": null,
  "linkedJob": null
}
```

## Update Enquiry Status (Admin)

```http
PATCH /api/v1/enquiry/:enquiryId
Authorization: Bearer <admin_clerk_jwt>
Content-Type: application/json
Origin: https://aotf.in
```

**Permission required**: `canUpdateEnquiryStatus`

**Request Body**:
```json
{
  "currentStatus": "in_progress",
  "lastActionNote": "Called consumer — interested in Math tutor"
}
```

**Response 200**: Updated enquiry document.

**Side effects**:
- Creates an `EnqStatus` document (audit trail)
- Updates `EnquiryLedger` and Google Sheet

## Enquiry Sources

The `source` field on an enquiry tracks where the consumer came from:

| Source Code | Meaning |
|---|---|
| `website_form` | Submitted via the website enquiry form |
| `whatsapp` | Consumer messaged via WhatsApp |
| `phone_call` | Consumer called in |
| `referral` | Referred by an existing consumer |
| `social_media` | Instagram / Facebook DM |
| `walk_in` | In-person visit |

Sources are configured in `lib/validations/forms.ts` → `sourceLists` array and are shared by `Post`, `Job`, and `Enquiry` models.
