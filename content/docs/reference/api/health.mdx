# Health Check API

`GET /api/health`

Returns the current state of the server and database connection. Useful for uptime monitors, CI smoke tests, and manual verification after deployment.

## Request

```http
GET /api/health HTTP/1.1
Host: aotf.in
```

No authentication required. No request body.

## Response

### 200 — Healthy

```json
{
  "status": "ok",
  "db": "connected",
  "uptime": 3600,
  "version": "0.0.1",
  "timestamp": "2026-06-06T12:00:00.000Z"
}
```

### 503 — Degraded (DB Unreachable)

```json
{
  "status": "degraded",
  "db": "disconnected",
  "uptime": 3600,
  "version": "0.0.1",
  "timestamp": "2026-06-06T12:00:00.000Z"
}
```

## Response Fields

| Field | Type | Description |
|---|---|---|
| `status` | `"ok"` \| `"degraded"` | Platform health status |
| `db` | `"connected"` \| `"disconnected"` | MongoDB connection state |
| `uptime` | `number` | Seconds since the Node.js process started |
| `version` | `string` | App version from `package.json` |
| `timestamp` | `string` | ISO-8601 UTC timestamp of this response |

## Implementation Notes

- Returns **HTTP 200** when DB is connected, **HTTP 503** when DB is unreachable
- Uses `export const dynamic = "force-dynamic"` — never cached by Vercel CDN
- Calls `dbConnect()` internally — this wakes a paused MongoDB Atlas M0 cluster
- First call after a cold period may take **5–15 seconds** while MongoDB resumes

## Usage Examples

### Vercel Health Check (uptime monitor)

Configure your uptime monitor (UptimeRobot, BetterUptime, etc.):

```
URL:              https://aotf.in/api/health
Method:           GET
Expected status:  200
Expected keyword: "ok"
```

### curl

```bash
curl -s https://aotf.in/api/health | jq .
```

### npm script (CI smoke test)

```bash
# Check health after deploy
curl -f https://aotf.in/api/health || exit 1
```
