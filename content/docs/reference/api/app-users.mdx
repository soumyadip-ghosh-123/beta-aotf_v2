# Admin App Users API

The admin **Users** page (`/admin/users`) reads from MongoDB via these routes — not from Clerk directly. Requires `canManageUsers` (or `super_admin`).

**Source files**: [`app/api/admin/app-users/route.ts`](file:///d:/aotf_v2/app/api/admin/app-users/route.ts), [`app/api/admin/app-users/[userId]/status/route.ts`](file:///d:/aotf_v2/app/api/admin/app-users/[userId]/status/route.ts)

## List Users

```http
GET /api/admin/app-users
Authorization: Bearer <admin_clerk_jwt>
```

**Query parameters**:

| Param | Description |
|---|---|
| `role` | `teacher` or `candidate` (maps to `teacher_candidate` in DB) |
| `status` | `active`, `blocked`, `deleted`, or `all` |
| `search` | Filter by username or email |
| `page` | Page number (default 1, 10 per page) |
| `bundle` | `1` — return both roles' first page + global summary |
| `sync` | `1` — run `syncClerkAppUsers()` before listing (backfill missing Mongo records from Clerk) |

**Response 200** (paginated):
```json
{
  "users": [
    {
      "_id": "...",
      "clerkId": "user_...",
      "username": "teacher1",
      "email": "teacher@example.com",
      "role": "teacher",
      "status": "active",
      "onboardingCompleted": true,
      "profileUrl": "/u/teacher1",
      "verifyUrl": "/verify/AOTF-T-TEACHER1"
    }
  ],
  "pagination": { "page": 1, "limit": 10, "total": 142, "totalPages": 15 }
}
```

Verify URL format: `/verify/AOTF-T-{USERNAME}` for teachers, `/verify/AOTF-C-{USERNAME}` for candidates.

## Create User

```http
POST /api/admin/app-users
Authorization: Bearer <admin_clerk_jwt>
Content-Type: application/json
```

Creates a Clerk user (no password) and seeds MongoDB via `createAppUser()` in [`lib/services/app-user.service.ts`](file:///d:/aotf_v2/lib/services/app-user.service.ts).

**Request body**:
```json
{
  "name": "Full Name",
  "email": "user@example.com",
  "phone": "9876543210",
  "role": "teacher"
}
```

`role`: `"teacher"` or `"candidate"`.

## Update User Status

```http
PATCH /api/admin/app-users/:userId/status
Authorization: Bearer <admin_clerk_jwt>
Content-Type: application/json
```

**Request body**:
```json
{ "status": "blocked" }
```

Valid values: `"active"`, `"blocked"`, `"deleted"`.

**Side effects**:
- Block/delete: revokes all Clerk sessions
- Updates Clerk `publicMetadata.accountStatus`
- Logs `BLOCK_USER` / `UNBLOCK_USER` activity

## Sync Behavior

The Users page calls `GET /api/admin/app-users?bundle=1&sync=1` on load. This ensures Clerk accounts created by migration (or manually) have matching `User` + `Profile` documents before the list renders.

For CLI backfill, use `pnpm sync:clerk-users` — see [Seed Teachers & Candidates](/docs/how-to/seed-users).
