## Root Directory

```
aotf_v2/
├── app/                    # Next.js App Router pages & API routes
├── assets/                 # Static assets (images, icons)
├── calendar/               # Calendar-related utilities
├── components/             # Reusable React components
├── config/                 # App configuration (site, fonts)
├── constants/              # Shared constant values
├── content/                # Fumadocs MDX documentation
├── cookies/                # Cookie utilities
├── hooks/                  # Custom React hooks
├── lib/                    # Core business logic
├── public/                 # Public static files
├── scripts/                # CLI scripts (seeding, testing)
├── styles/                 # Global CSS
├── types/                  # Shared TypeScript type definitions
├── proxy.ts                # Clerk middleware + route protection
├── instrumentation.ts      # Sentry + DB warmup instrumentation
├── source.config.ts        # Fumadocs MDX source configuration
├── next.config.js          # Next.js configuration
├── tailwind.config.js      # Tailwind CSS configuration
└── package.json            # Dependencies & scripts
```

## `/app` — Application Routes

The `app/` directory follows the Next.js App Router convention:

```
app/
├── (auth)/                 # Auth-related pages (sign-in, sign-up)
├── (static)/               # Static marketing pages (about, terms, etc.)
├── admin/                  # 🔒 Protected admin dashboard
│   ├── dashboard/          # Admin home
│   ├── admins/             # Admin management
│   ├── enquiries/          # Enquiry management
│   ├── feedbacks/          # Feedback management
│   ├── tuitions/           # Tuition post management
│   ├── jobs/               # Job post management
│   ├── users/              # User management
│   ├── invoices/           # Invoice management
│   ├── payments/           # Payment tracking
│   ├── ads/                # Ad management
│   ├── reviews/            # Review management
│   ├── settings/           # Admin settings
│   ├── login/              # Admin login page
│   └── change-password/    # Forced password change
├── api/                    # Backend API routes
│   ├── v1/                 # Versioned REST endpoints
│   │   ├── admin/          # Admin management APIs
│   │   ├── posts/          # Tuition post APIs
│   │   ├── jobs/           # Job listing APIs
│   │   ├── enquiry/        # Enquiry APIs
│   │   ├── feedback/       # Feedback APIs
│   │   ├── payments/       # Payment APIs
│   │   ├── profile/        # User profile APIs
│   │   ├── users/          # User management APIs
│   │   ├── webhooks/       # Clerk webhook handler
│   │   └── ...             # Additional endpoints
│   └── search/             # Fumadocs search endpoint
├── docs/                   # Documentation pages (Fumadocs)
├── enquiry/                # Public enquiry form
├── invoices/               # Public invoice view
├── jobs/                   # Public job listings
├── onboarding/             # User onboarding flow
├── posts/                  # Public tuition listings
├── u/                      # User profile pages (/u/[username])
└── verify/                 # Verification pages
```

## `/lib` — Core Business Logic

The `lib/` directory contains all shared logic, separated by concern:

```
lib/
├── models/                 # Mongoose schemas & models
│   ├── User.ts             # User accounts
│   ├── Profile.ts          # User profiles (education, skills)
│   ├── Admin.ts            # Admin accounts with RBAC
│   ├── Post.ts             # Tuition posts
│   ├── Job.ts              # Job/project listings
│   ├── Application.ts      # Gig applications
│   ├── ApplicationEvent.ts # Application lifecycle events
│   ├── Enquiry.ts          # Customer enquiries
│   ├── Feedback.ts         # User feedback
│   ├── Invoice.ts          # Generated invoices
│   ├── Payment.ts          # Payment records
│   ├── CalendarEvent.ts    # Calendar events (write-through)
│   ├── TodoEvent.ts        # Todo/task events
│   ├── AuditLog.ts         # Admin action audit trail
│   ├── LoginAttempt.ts     # Login attempt tracking
│   └── ...                 # Additional models
├── services/               # Business logic layer
│   ├── admin.service.ts    # Admin CRUD operations
│   ├── application.service.ts  # Application lifecycle
│   ├── post.service.ts     # Post management
│   ├── job.service.ts      # Job management
│   ├── enquiry.service.ts  # Enquiry processing
│   ├── email.service.ts    # Email templates & sending
│   ├── clerk.service.ts    # Clerk API interactions
│   ├── calendar-event.service.ts # Calendar sync
│   ├── invoice.service.ts  # (via postLedger.service)
│   └── ...                 # Additional services
├── validations/            # Zod schemas for request validation
│   ├── forms.ts            # Large shared form schemas
│   ├── post.ts             # Post validation
│   ├── job.ts              # Job validation
│   ├── enquiry.ts          # Enquiry validation
│   └── ...                 # Additional schemas
├── utils/                  # Utility functions
│   ├── ensure-user.ts      # Middleware user resolution
│   ├── formatCurrency.js   # INR formatting
│   ├── pdfGenerator.js     # PDF creation
│   └── share.ts            # Social sharing utilities
├── db.ts                   # MongoDB connection (retry, pooling)
├── api-utils.ts            # API helpers (CSRF, error handling)
├── errors.ts               # Custom error classes
├── rate-limit.ts           # In-memory rate limiter
├── source.ts               # Fumadocs page source
└── layout.shared.tsx       # Shared layout options
```

## Key Configuration Files

| File | Purpose |
|------|---------|
| `next.config.js` | Next.js config with Fumadocs MDX, Sentry, security headers |
| `proxy.ts` | Clerk middleware — auth, admin guards, onboarding gates |
| `source.config.ts` | Fumadocs MDX source definition |
| `config/site.ts` | Site-wide configuration (contact, SEO, payments, features) |
| `config/fonts.ts` | Font configuration |
| `instrumentation.ts` | Server startup hooks (Sentry, DB warmup) |
| `tsconfig.json` | TypeScript compiler options |
| `tailwind.config.js` | Tailwind CSS customisation |
| `eslint.config.mjs` | ESLint rules |
| `vercel.json` | Vercel deployment settings |
